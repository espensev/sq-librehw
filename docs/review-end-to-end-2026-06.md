# End-to-End Review — June 2026 (multi-agent sweep)

**Scope:** the full custom surface since fork point `abfc4f5` plus the hot paths it sits on — plot rendering, the 1 Hz MainForm update loop, GraphInputsForm, HttpServer (data.json / /metrics / /Sensor), CSV Logger, lib sensor core, NvidiaGpu identifier fix, tray/gadget paint, Aga.Controls tree, and a dedicated cross-cutting threading audit.

**Method:** 9 parallel area reviewers + build check; every finding adversarially verified (3-lens panel for high severity, majority vote). 98 agents total.

**Results:** 66 findings — **48 confirmed, 2 refuted, 16 unverified** (verification agents for ~3 areas hit the session usage limit; their findings are marked `unverified` below — reviewer confidence was high and several share root causes with independently confirmed findings). Build: **0 warnings, 0 errors** across all TFMs.

**Contract note:** data.json shape and CSV Identifier columns are external contracts (downstream consumer). Every suggested fix below was screened for contract safety; contract-affecting items are called out explicitly.

## Priority shortlist

1. **Plot hot path** (high/confirmed): every plotted point is re-transformed and re-copied every tick — up to N×21,600 point transforms/sec on the UI thread; the long-window spec (`docs/feature-long-window-graph-rendering.md`) is still Draft/unimplemented. Fix: absolute-time anchored incremental point lists + `Decimator.Decimate`.
2. **Worker-thread UI mutation family** (high/confirmed): sensor activation during `hardware.Update()` fires `SensorAdded` on the BackgroundWorker thread → Node tree, TreeViewAdv, PersistentSettings, and OxyPlot model all mutated off the UI thread. Same root cause behind intermittent **data.json 500s** (contract surface). Fix: marshal add/remove handling to the UI thread + snapshot/lock tree traversal in HttpServer.
3. **NVIDIA pin-voltage re-key** (high/confirmed/contract): `/voltage/1..5` now mean different physical pins; needs a one-time downstream remap table, and a forced log rotation if Daily CSV ever enabled.
4. **Sensor.Values unsynchronized** (confirmed): live `List<SensorValue>` shared with /metrics (full-copy `Reverse()`) and OxyPlot. Fix: per-sensor lock + snapshot getter.
5. **HTTP shutdown 5 s UI freeze** (confirmed): `Stop()` ordered after the 5 s `Wait` — every app exit with web server on freezes ~5 s. One-line reorder.
6. **GET-reachable fan control writes** (confirmed/security): `action=Set` works via plain GET → drive-by CSRF from any web page; make mutations POST-only.
7. **Logger PerSession rotation storm** (confirmed): legal interval combinations create one file per row with quadratic `File.Exists` scanning; plus Daily-mode truncation data-loss path and exceptions silently killing plot invalidation.
8. **CorsairPsu precedence bug** (confirmed): `acc + sensor.Value ?? 0f` parses as `(acc + sensor.Value) ?? 0f` — 'Total Output' collapses on a null rail read; wrong value flows into data.json/CSV.

Then the quick wins: tray icon full GDI rebuild + shell IPC per tick even when unchanged; `StringFormat.GenericTypographic` native leak per sensor per tick in SensorGadget; `GetFanCoolersStatus` called twice per tick; reflection-cache in Aga `BindableControl`; NaN poisoning the 4-sample average; `RemoveAt(0)` O(n²) on window shrink; data.json ~1 MB/s GC garbage at 1 Hz polling (use `SerializeToUtf8Bytes` + `GetBuffer`); /metrics O(n²) string concat.

**Refuted (verification working as intended):** the "512 KB LOH per static request" claim — dead code because `ServeResourceFileAsync` uses the stale resource prefix `LibreHardwareMonitor.Resources.` (actual: `LibreHardwareMonitor.Windows.Forms.Resources.*`), so **the embedded web UI 404s entirely** — itself a latent bug if the web UI is ever wanted. Also refuted: per-sensor Min/Max reset "bypassing" Computer._lock via the bulk route (the bulk route does serialize correctly; only the per-sensor route bypasses).

**Follow-up gaps identified by the completeness critic:** dual-TFM/manifest/DPI surface (Program.cs, both manifests, csproj rewrites — zero coverage); TestScripts Python consumers never validated against the modified contract (LiquidCool.py pins identifiers the NVIDIA fix re-keyed — it drives fan control); embedded web UI as data.json consumer (+ stored-XSS question, currently moot while resources 404); no single-instance guard (two instances interleave writes into the same CSV = worst contract corruption vector) and settings persisted only at exit; GadgetWindow.cs layered-window repaint pipeline; five spec docs unchecked for drift.

## AREA: plot-render
SUMMARY: The custom plot work (grid density, local-time axis labels, batched plot-selection rebuilds) is carefully written — guarded property assignments avoid OxyPlot re-tick churn, bulk plot changes coalesce into one SetSensors call, and the real-time label formatter matches its spec including date-boundary and sub-minute cases. The structural weakness is the inherited hot path the fork has not yet addressed: relative-age X coordinates force OxyPlot to re-enumerate and re-copy every retained point of every plotted series once per second (default retention is 24 h, so up to ~21,600 points per sensor) with no decimation, exactly the problem docs/feature-long-window-graph-rendering.md (still Draft, unimplemented) describes. No external-contract (data.json/CSV) risk exists in this area; all suggested fixes are display-only.

OVERALL: Coverage of the assigned areas is strong: every hot path named in the brief (1 Hz tick, plot render, HTTP responses, CSV writes, tray paint) has substantive findings, the two external contracts were examined from the server/producer side, and threading was cross-cut properly.
